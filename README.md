# claude-doctor

Diagnose your Claude Code sessions. Analyzes `~/.claude/` transcripts for quality signals, behavioral anti-patterns, and sentiment drift. Generates AGENTS.md rules from your history.

## Install

```bash
npm i -g claude-doctor
```

## Usage

```bash
# Check health of your latest session
claude-doctor

# Check a specific project
claude-doctor -p myproject

# Check a specific session file
claude-doctor -s ~/.claude/projects/.../SESSION_ID.jsonl

# Analyze all sessions across all projects
claude-doctor --all

# Generate AGENTS.md rules from your history
claude-doctor --rules

# Save model to .claude-doctor/
claude-doctor --save

# JSON output
claude-doctor --json
```

## What it detects

### Structural signals

| Signal | What it catches |
|---|---|
| `edit-thrashing` | Same file edited 5+ times in one session |
| `error-loop` | 3+ consecutive tool failures |
| `excessive-exploration` | Read-to-edit ratio above 10:1 |
| `restart-cluster` | Multiple sessions started within 30 minutes |
| `high-abandonment-rate` | Most sessions end with fewer than 3 messages |

### Behavioral signals

| Signal | What it catches |
|---|---|
| `correction-heavy` | 20%+ of user messages are corrections |
| `keep-going-loop` | User repeatedly says "keep going" / "continue" |
| `repeated-instructions` | User says the same thing twice (Jaccard >60%) |
| `negative-drift` | Messages get shorter and more corrective over time |
| `rapid-corrections` | User responds within 10s of agent output |
| `high-turn-ratio` | User sends 1.5x+ messages per agent response |

### Lexical signals

AFINN-165 sentiment scoring with custom agent-specific tokens.

## Generating AGENTS.md rules

```bash
claude-doctor --rules
```

```markdown
## Auto-generated rules

Based on analysis of 838 sessions. Generated by claude-doctor.

- Read the full file before editing. Plan all changes, then make ONE complete edit.
- After 2 consecutive tool failures, stop and change your approach entirely.
- When the user corrects you, stop and re-read their message.
- Complete the FULL task before stopping.
```
